    But we are showing chat messages, not from a file. box_chat.php directly, we display messages individually in a nice way. I will explain it to you There are several ways to display messages in chat box, but not all methods display messages correctly, some methods caused more RAM space, because of setInterval, it fetches data every second, and puts it into chatbox, maybe. The messages are large and this causes problems and heaviness in the browser. I've created a great way to display messages in a smooth and light way on the browser, let's quickly start taking a look at this line of code, when we get into the chat box. This code fetches all messages and puts them in the chatbox in the first second, after that it stops fetching all messages, and fetches the last message sent, unlike the method of fetching all messages every second, we retrieve messages using setinterval, depending on the value of the variable last_msg_chat_id but what is the value of this variable? The value of this variable is the ID of the last message in the chat box. For example chat_id = 10; And what do we do with it, we tell the query to show the message that has an id greater than this id, this is useful it shows us only recent messages without any problems and it is light on the browser also quickly, and it reflects how to fetch all chat messages every second, instead, we used More easy way, hi, it fetches the last message that was just sent, directly and puts it in the chat box by append function
