Install and use websocket in laravel 10 with php 8.0 and above

2023.06.30 by Murad Shaker

When laravel 10 is released, you may have difficulty installing WEBskoted 2.0 in laravel 10 because laravel was installed with php 8.0 or later. However, WEBskoted 2.0 is not supported with php 8.0, and in this case a conflict occurs. In this article, you will learn how to install the appropriate WEBskoted. With laravel 10, first let me tell you something that currently laravel 10 supports WEBskoted version 1.14.0 so we will install 1.14.0 WEBskoted

Let's Begin

Now let's install the latest version of Laravel on the computer, type this command

 composer create-project laravel/laravel project-name 

As I mentioned above that the latest version of laravel identifier requires php version 8.0 or above so make sure if you are installing php version 8.0 or above on your computer

After installing Laravel, now select the project folder in the command prompt as shown below

To install the appropriate version of Websockets in Laravel
Type this command

 composer require beyondcode/laravel-websockets -w 

Don't forget to put a -w at the end of this command
The Websockets package automatically comes with a migration file. This migration is used to store or track statistical information, for example, how many messages are sent per day, the number of errors, the number of connections, and more. It is very useful if we want to know what is going on in the current  Websockets server.
You can move the migration file contained in the Websockets package to the database/migrations directory for us to migrate to the database
This is done by running this command at the command prompt

 php artisan vendor:publish --provider="BeyondCode\LaravelWebSockets\WebSocketsServiceProvider" --tag="migrations" 

The migration file will be moved to the database/migrations directory, as shown in this image

Next, run the  php artisan migrate  command to send the package's migration file to the database
After running this command, the statistics table became side by side within your database tables
As shown in the picture

Now you should run this command

 php artisan vendor:publish --provider="BeyondCode\LaravelWebSockets\WebSocketsServiceProvider" --tag="config" 

This command displays the main Websockets package file in the project-Name/config directory
As shown in the picture

 

Now it's time to install the Pusher package.

Type this command in the command prompt

 composer require pusher/pusher-php-server 

You may say what is the benefit of this package: This package is used to provide current real-time events between the server and the client, where it is used with WebSockets to facilitate the process.
Now go to the  .evn  file
Put the word "pusher" next to BROADCAST_DRIVER=

Also set ID, key, and SECRET to PUSHER
This way

 PUSHER_APP_ID=123456 
 PUSHER_APP_KEY=goodfonyerv 
 PUSHER_APP_SECRET=lskobertndww 
 PUSHER_APP_CLUSTER=mt1 

You can choose different numbers and letters

This is the details of PUSHER such as the  unique identifier  and  key  which is used to authenticate the client when establishing a websocket connection with pusher and the  SECRET  which is used to authenticate the server and authorization when we want to interact with the PUSHER package eg triggering events and authenticating channels

Don't worry, I will explain to you the lines inside this file
first line
  'port' => env('LARAVEL_WEBSOCKETS_PORT', 6001),

This line specifies the port number that is being listened to for example, if John  wants to listen to you, in this case he sets the number 6001 to listen to you because it is your port number , If you do not set a port number to LARAVEL_WEBSOCKETS_PORT in the  .env  file

as in the picture

In this case the port number will be set automatically based on the number 6001 specified in the line
You may say what is the use of this port number

: Let me explain to you, for example, you have a live chat system that contains many users, and you use the Websockets package in your application. In this case, now each user uses a unique port number, and this is useful when performing a specific action to a specific user, which provides us with access to the right person By specifying its port number, knowing that the Websockets server will create a unique port number for each person

I'm going to explain what the elements inside the apps array mean

  1. The first line: means to set the  id  of the pusher in the  .evn  file as a value to the "id" key of the apps array
  2. The second line: means to set the name of the pusher in the  .evn  file as a value to the "name" key of the apps array
  3. The third line: means to set the the key of the pusher in the  .evn file as a value to the "key" key of the apps array
  4. Line 4: Set the secret of the pusher in the  .evn  file as a value to the "secret" key of the apps array
  5. Line 5: Since null is assigned to the "capacity" key, this means that no capacity is specified for this application
  6. Sixth line: Since false is set to the "enable_client_messages" key, this means that client messages are disabled for this application (the client cannot send a message to the server), but (the server can send a message to the client)
  7. The previous line: Since true is set to the "enable_statistics" key, this means that we have now allowed the application or the Websockets package to monitor and record the statistics of the current Websockets server. I mentioned what the statistics are above, you can read them

 

Where the "app_provider" key defines important or essential websockets package applications (id, key, secret, etc.), by calling them via this line BeyondCode\LaravelWebSockets\Apps\ ConfigAppProvider:: class , where these applications are used inside the  websockets .php  file in order to set a value for them or update them and so on

This model is used to store statistics


'logger' => BeyondCode\LaravelWebSockets\Statistics\Logger\HttpStatisticsLogger::class,
This line is used to record statistics


'interval_in_seconds' => 60, In this line you can specify the time interval in seconds that you want to record the statistics, so if you choose 60 in this case every 60 seconds will record the statistics


  'delete_statistics_older_than_days' => 60, 
In this line, you can specify the days after which the statistics are deleted from the database


'perform_dns_lookup' => false,
This line is used to determine whether DNS should perform a lookup or not when sending requests to the statistics registrar. If you are using local hosting, you should choose false (as setting it to false will send all requests to 127.0.0.1). If you are running the project on real hosting, you should choose true to send the requests to the current valid IP address.

It determines the channels to which the event should be broadcast: Channels are used to specify clients based on the name of the channel specified to broadcast events to them

There are two types of channels
public
private

Public channels:  available for any person or client to listen to.

Private channels:  Available only to subscribed customers or a specific group of people
